# 07 · 提交得分记录

> 记录每个提交产物在 A 榜的**实际平台得分**，用于反推单任务原始分、验证提分方向。
> 每次平台出分后**追加一行**，不要删旧行（对照实验靠历史行）。

## 评分模型

平台总分 = **0.4 × s1 + 0.6 × s2**
- s1 = Task1 聚类原始分（`pattern_reco.csv`）
- s2 = Task2 资金识别原始分（`predict_result.csv`，加权 F1，仅 `capital_type` 计分）

单任务提交时，未跑的那份 CSV 只写表头（`--task N`）；若平台对空表判 0，则该次总分只反映另一任务，可据此反推单任务原始分。

## 关键假设与待验证

反推依赖三条假设，**(C) 尚未验证**：
- **(A) 空 task1 判 0 分** —— 已侧证：仅 task2 得 0.1727（非报错、非 0），说明平台接受空表头 CSV。
- **(B) 两次提交 task2 结果一致** —— 成立：同一 `v1` 产物。
- **(C) 评分严格线性可加，s1/s2 互不影响** —— **未验证**。天池部分赛题 task 分是组内归一化/排名分，未必线性。
  - 验证法：单独提交 `--task 1`，若线性成立应得 **0.4 × 0.703 ≈ 0.281**。
  - 吻合 → 模型可信，s1/s2 反推值成立；不吻合 → 评分非线性，反推作废，需换测法。

## 分布探针（全押单类反解真实占比）

全押同一类 A 时，加权 F1 有闭式解 `weightedF1 = 2·p_A²/(1+p_A)`（p_A=该类真实占比）。
提交为「task1 空 + task2 全押 A」，故 **s2 = 平台总分 / 0.6**，再反解 p_A。

- **散户**：总分 0.1560 → s2=0.260 → **p_散户 ≈ 0.431**。
- **游资**：总分 0.1306 → s2=0.218 → **p_游资 ≈ 0.389**。
- **量化**：总分 0.0266 → s2=0.044 → **p_量化 ≈ 0.160**。
- **Σp = 0.980 ≈ 1** ✅ —— 三针独立测量之和几乎为 1，验证评分为标准加权 F1、线性假设 (C) 成立。
- **归一化真实分布（⚠️ 仅 20260706 单日）：散户 44% / 游资 40% / 量化 16%**（99 只配额：44 / 39 / 16）。
  > **此分布只代表 20260706 这一天。** 其他交易日资金结构未必相同，44/40/16 **不可作为固定先验跨日复用**；
  > 若用配额分配，每个交易日的配额都应基于当日数据重新估计，否则会引入分布偏差、反噬加权 F1。

## 当前判断（探针已定死真实分布）

- **20260706 单日真实分布：散户 44% / 游资 40% / 量化 16%**（反解 Σp=0.98≈1，线性假设 (C) 侧证通过）。
  ⚠️ 仅当日有效，其他交易日未必如此，勿把该占比当固定先验套用。
- **当前 pipeline 连"按真实占比随机猜"都不如**：随机按占比的加权 F1 期望 = Σp² ≈ **0.378**，
  而当前 s2=**0.288**。**负智能** —— 现有 argmax+校准的组合把分类做反了。
- **量化被严重高估 + calibrate 是负优化铁证**：基准预测量化 28% vs 真实 16%（几乎翻倍）；
  而 `calibrate` 下限 tmin=0.22 > 真实 16%，**强行把量化撑到 ≥22 个，主动制造误判**。
- **立即可提分动作**：弃用 argmax + `[0.22,0.45]` 校准，改为**按真实占比精确配额**
  （99 只 → 游资 39 / 散户 44 / 量化 16），用现有三因子分数排序切分。
  - 期望 **≥0.378**（比现在 +0.09）；
  - 若结果 **远超 0.378** → 三因子分数有真实区分力，值得深挖优化；
  - 若结果 **≈0.378** → 分数是噪声，配额只吃了分布红利，需重做特征/模型。
- **s1≈0.703 健康**，task2 权重 0.6 仍是主战场。
