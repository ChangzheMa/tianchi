一、思考题
（一）如何优化 Baseline 方案？
特征工程优化（最高优先级，投入产出比最高）
● 盘口微观结构特征：
  ○ 订单流不平衡（OFI）：计算连续快照间买卖挂单的变化量，是学术界公认的强特征。公式：OFI = Δbid_volume - Δask_volume。
  ○ 深度加权价格：根据盘口深度计算加权平均买卖价格，比简单的最优价差更精确。
  ○ 价差动态：买卖价差的均值、标准差、变化率，反映流动性变化。
  ○ 利用 bidaskrate/bidaskdifference：当前训练数据中这两个字段为 0，但 A/B 榜可能有值，届时可提取为特征。
● 时间序列特征：
  ○ 滑动窗口统计：过去 N 个快照的均值、标准差、趋势斜率。
  ○ 自相关特征：价格/成交量的自相关系数。
● 量价关系特征：
  ○ 量价背离：价格上涨但成交量下降，或反之。
  ○ VWAP 偏离度：当前价格与成交量加权平均价的偏离。
● 撤单行为特征（当前因快照数据缺失而置 0）：
  ○ 有逐笔撤单数据时，补充撤单率、快速撤单占比、撤单间隔变异系数等。
模型优化
● Task 1 聚类：
  ○ 尝试 DBSCAN、HDBSCAN、GMM 等密度/概率聚类方法。
  ○ 使用 DTW 距离或 Wasserstein 距离替代欧氏距离，更符合赛题评估标准。
  ○ 对时间序列构建 TimeSeriesKMeans（基于 DTW）。
● Task 2 分类：
  ○ 引入 XGBoost、LightGBM、CatBoost 等梯度提升树模型，替代纯规则打分。
  ○ 基于聚类标签构建伪标签，训练有监督分类器。
  ○ 多任务学习：同时预测参与者类型和意图，共享底层特征表示。
伪标签/规则质量提升
● 当前规则基于金融领域先验知识，可进一步细化和量化阈值。
● 利用 A 榜反馈的 F1 分数反向优化规则阈值。
● 半监督学习：少量标注 + 大量未标注，通过自训练提升标签质量。
引入外部数据
● 基本面数据：市值、行业、PE/PB 等，不同行业的参与者结构可能不同。
● 多日历史数据：利用多日 Level-2 数据构建更丰富的训练集。
● 新闻/舆情数据：赛题允许使用盘后非实时信息进行事后验证与修正。
多股票适配
● 当前训练数据仅 1 只股票，A/B 榜有 100 只股票。需考虑：
  ○ 不同市值、不同行业的股票特征分布差异。
  ○ 对特征做市值/行业中性化处理。
  ○ 聚类时考虑跨股票的模式一致性。
（二）详细的提分教程
以下是具体的、可操作的提分路径，按照难度和预期收益排序：

1、提分路径一：盘口微观结构特征
核心思路：当前 Baseline 只用了首条快照的盘口特征（spread/book_imbalance）和全天统计量。盘口微观结构中的"订单流不平衡（OFI）"是学术界公认的强特征。
实现方法：
# 在 extract_all_feature 中，对每条快照计算 OFI
# OFI = (bid_volume_t - bid_volume_t-1) - (ask_volume_t - ask_volume_t-1)
# 正值 → 买方挂单在增加，买盘压力大；负值 → 卖方挂单在增加，卖盘压力大

group['ofi'] = (group['totalbidvolume'].diff() - group['totalaskvolume'].diff())
feature['ofi_mean'] = group['ofi'].mean()
feature['ofi_std'] = group['ofi'].std()
feature['ofi_positive_ratio'] = (group['ofi'] > 0).sum() / len(group)
为什么有效：OFI 反映的是"新挂单"的方向——游资拉升时，会不断在买盘挂大单吸引跟风盘，OFI 会持续为正；量化 T0 套利时，买卖挂单交替变化，OFI 在零附近波动。

2、提分路径二：撤单行为特征补充
核心思路：当前 Baseline 中 CB 特征全为 0，因为快照数据没有撤单明细。但如果你能获取到逐笔撤单数据（赛题数据可能包含），撤单行为是区分游资和量化最有效的特征之一。
实现方法：
# 假设有 cancels 数据表，包含 cancel_time, cancel_volume, cancel_direction
# 快速撤单占比：挂单后 < 1 秒内撤单的占比
fast_cancel = cancels[cancels['cancel_interval_ms'] < 1000]
feature['cb_fast_cancel_ratio'] = len(fast_cancel) / len(cancels)

# 买卖撤单分化：买方撤单 / 卖方撤单
feature['cb_buy_cancel_ratio'] = len(cancels[cancels['direction'] == 'buy']) / len(cancels)
feature['cb_sell_cancel_ratio'] = len(cancels[cancels['direction'] == 'sell']) / len(cancels)

# 撤单金额占比
feature['cb_cancel_amount_ratio'] = cancels['cancel_amount'].sum() / total_tick_amount
为什么有效：量化机构频繁挂单撤单测试市场深度，快速撤单率远高于游资；游资撤单较少，以成交为目的。

3、提分路径三：使用 XGBoost/LightGBM 替代规则打分
核心思路：当前 Task 2 使用纯规则打分，主观性强。利用聚类标签作为伪标签，训练有监督模型。
实现方法：
from xgboost import XGBClassifier

# Step 1: 用当前规则生成伪标签
df_feature['pseudo_label'] = df_feature.apply(calc_score, axis=1)

# Step 2: 训练 XGBoost 分类器
X = df_feature[all_feature_cols].values
y = (df_feature['pseudo_label'] == '游资').astype(int)

model = XGBClassifier(
    n_estimators=200,
    max_depth=5,
    learning_rate=0.05,
    random_state=42
)
model.fit(X, y)

# Step 3: 预测 + 概率校准
proba = model.predict_proba(X)[:, 1]
df['capital_type'] = np.where(proba > 0.5, '游资', '量化机构')
为什么有效：XGBoost 能自动学习特征间的非线性交互关系，比人工规则打分更灵活；同时能输出概率，可用于置信度评估。

4、提分路径四：动态阈值优化
核心思路：当前聚类模式匹配的阈值是硬编码的（如 oss_mega_amount_pct > 0.12）。可以通过网格搜索或贝叶斯优化找到最优阈值。

5、提分路径五：时间序列建模
核心思路：当前方案将全天数据聚合为单样本，丢失了时序信息。可以尝试 LSTM/Transformer 直接对时序建模。
实现方法：

6、提分路径六：大模型解决方案
核心思路：利用大语言模型（LLM，如 GPT-4、Claude、DeepSeek、Qwen 等）的语义理解、推理和代码生成能力，从多个维度加速和优化解题过程。大模型在金融数据分析中的优势在于：能理解金融领域的业务逻辑、能生成高质量的特征工程代码、能进行多维度推理分析。

实现示例（基于 LangChain/LangGraph）：
from langchain.agents import AgentExecutor, create_openai_tools_agent
from langchain.tools import tool
from langchain_openai import ChatOpenAI

# ── 定义各 Agent 的工具 ──

@tool
def analyze_data_schema(data_path: str) -> str:
    """分析数据文件的字段结构和统计特征"""
    import pandas as pd
    df = pd.read_excel(data_path, engine='openpyxl')
    return f"数据维度: {df.shape}, 字段: {list(df.columns)}, 缺失率: {df.isnull().mean().to_dict()}"

@tool
def suggest_features(data_description: str) -> str:
    """基于数据描述推荐特征工程方案"""
    # 调用 LLM 推理
    llm = ChatOpenAI(model="gpt-4")
    response = llm.invoke(f"基于以下数据描述，推荐 15 个金融特征:\n{data_description}")
    return response.content

@tool
def evaluate_clustering(feature_matrix, labels) -> str:
    """评估聚类效果并给出优化建议"""
    from sklearn.metrics import silhouette_score, davies_bouldin_score
    sil = silhouette_score(feature_matrix, labels)
    db = davies_bouldin_score(feature_matrix, labels)
    return f"轮廓系数: {sil:.4f}, DB指数: {db:.4f}"

# ── 构建 Agent 协作流程 ──
# 1. Feature Engineer Agent 分析数据并生成特征
# 2. Model Builder Agent 选择模型并训练
# 3. Eval Analyst Agent 评估结果并诊断
# 4. 迭代优化直到满足指标要求
# 5. Report Writer Agent 生成最终方案报告
为什么有效：多智能体协作将复杂任务分解为专业化子任务，每个 Agent 专注于自己的领域，通过协同工作实现比单一 Agent 更好的效果。这种方式特别适合需要多维度专业知识的金融 AI 竞赛。

附录1：游资 vs 量化机构的交易特征对比
维度	游资	量化机构
交易风格	激进、追涨杀跌	稳健、系统化
订单特征	大单为主、集中出击	小单为主、分散下单
价格波动	高波动、拉抬/砸盘	低波动、避免影响价格
交易频率	中低频、择机而动	高频、持续交易
盘口行为	制造假象、诱多/诱空	盘口均衡、隐藏意图
撤单行为	撤单较少、成交为主	撤单频繁、试探市场
时段偏好	集中时段出击	全天均匀分布
交易结构	单笔交易量大、离散度高	单笔交易量小、均匀分散

附录3： 评估指标说明
● 轮廓系数（Silhouette Score）：衡量聚类效果，值域 [-1, 1]，越接近 1 越好。综合类内紧密度和类间分离度。
● CH 指数（Calinski-Harabasz Index）：方差比准则，越高越好。
● DB 指数（Davies-Bouldin Index）：衡量类间相似度，越低越好（类间差异大、类内聚合好）。
● 加权 F1 Score：F1 = 2 × Precision × Recall / (Precision + Recall)，加权版考虑类别不平衡。
● Wasserstein 距离：衡量两个概率分布之间的"推土机距离"，对金融数据分布差异敏感。
● DTW 距离（Dynamic Time Warping）：衡量时间序列相似度，允许时间轴弹性对齐。
