# 05 · 提分思路与优化方向

> 来源：`进阶教程内容.txt`。按「投入产出比」排序，附实现要点与生效原理。

## 一、特征工程优化（最高优先级，ROI 最高）

### 1. 盘口微观结构：订单流不平衡 OFI（学术界公认强特征）
```
OFI = Δbid_volume - Δask_volume   # 逐快照计算
```
- `ofi_mean / ofi_std / ofi_positive_ratio`。
- 原理：游资拉升时不断在买盘挂大单，OFI 持续为正；量化 T0 套利买卖挂单交替，OFI 在 0 附近波动。

### 2. 撤单行为特征（区分游资/量化最有效之一）
- 快速撤单占比（挂单后 <1s 撤单）、买卖撤单分化、撤单金额占比。
- 原理：量化频繁挂撤单测深度，快速撤单率远高于游资；游资撤单少、以成交为目的。
- 真实数据的**逐笔委托.csv 含撤单事件**，务必用起来（旧 main.py 中 CB 特征全为 0）。

### 3. 其他
- 深度加权价格、价差动态（均值/标准差/变化率）。
- 滑动窗口统计（过去 N 快照的均值/标准差/趋势斜率）、价格/量自相关。
- 量价背离、VWAP 偏离度。

## 二、模型优化

### Task1 聚类
- 尝试 DBSCAN / HDBSCAN / GMM 等密度/概率聚类。
- 用 **DTW 距离 / Wasserstein 距离**替代欧氏距离（更贴合评估标准）。
- 对时序构建 `TimeSeriesKMeans`（基于 DTW）。

### Task2 分类
- 引入 XGBoost / LightGBM / CatBoost 替代纯规则打分。
- 用规则生成**伪标签**训练有监督分类器（可输出概率做置信度）。
- 多任务学习：同时预测参与者类型与意图，共享底层表示。

### 阈值优化
- 当前模式匹配阈值硬编码。可用网格搜索 / 贝叶斯优化，在 [0.05, 0.50] 步长 0.01 搜，
  以聚类轮廓系数最大为目标。
- **用 A 榜每日反馈的 F1 反向优化规则阈值**。

### 时序建模
- 用 LSTM / Transformer 直接对每日快照序列建模，捕捉"先吸筹后拉升"的时序演变
  （需多股票多天数据）。

## 三、引入外部数据

- 基本面：市值、行业、PE/PB（不同行业参与者结构不同）。
- 多日历史 L2 数据构建更丰富训练集。
- 新闻/舆情（赛题允许用盘后非实时信息做事后验证与修正）。
- **多股票适配**：对特征做市值/行业中性化，考虑跨股票模式一致性。

## 四、大模型（LLM）方案

1. **辅助特征工程**（低门槛，推荐先做）：把数据 schema + 业务背景喂给 LLM，
   让它推荐能区分游资/量化的特征并生成代码。
2. **辅助模式解释与报告生成**（低门槛，推荐先做）：把聚类中心画像喂给 LLM 生成金融语义解释，
   批量生成方案报告初稿（报告占 B 榜可解释性评分）。
3. **辅助规则阈值优化**：让 LLM 基于市场微观结构理论给出分市值差异化阈值建议。
4. **多智能体协作**（进阶）：Feature Engineer / Model Builder / Eval Analyst / Report Writer
   四 Agent + Orchestrator（LangChain/LangGraph）。
5. **LLM 微调**（进阶）：有足够伪标签后，用 LoRA 微调 Qwen/DeepSeek 输出结构化分析理由。

> 推荐路径：先做「辅助特征工程」+「模式解释/报告生成」快速见效，
> 有经验和数据积累后再上「多智能体」「微调」。

## 五、附：游资 vs 量化 交易特征对比（判别先验）

| 维度 | 游资 | 量化机构 |
|------|------|----------|
| 交易风格 | 激进、追涨杀跌 | 稳健、系统化 |
| 订单特征 | 大单为主、集中出击 | 小单为主、分散下单 |
| 价格波动 | 高波动、拉抬/砸盘 | 低波动、避免影响价格 |
| 交易频率 | 中低频、择机而动 | 高频、持续交易 |
| 盘口行为 | 制造假象、诱多/诱空 | 盘口均衡、隐藏意图 |
| 撤单行为 | 撤单较少、成交为主 | 撤单频繁、试探市场 |
| 时段偏好 | 集中时段出击 | 全天均匀分布 |
| 交易结构 | 单笔量大、离散度高 | 单笔量小、均匀分散 |

## 六、评估指标速查

| 指标 | 方向 | 含义 |
|------|------|------|
| 轮廓系数 Silhouette | ↑ 越接近 1 越好 | 类内紧密 + 类间分离 |
| CH 指数 | ↑ 越高越好 | 方差比准则 |
| DB 指数 | ↓ 越低越好 | 类间相似度 |
| 加权 F1 | ↑ 越接近 1 越好 | 分类核心，考虑类别不平衡 |
| Wasserstein 距离 | ↑ 区分度越好 | 分布"推土机距离" |
| DTW 距离 | ↑ 区分度越好 | 时序弹性对齐相似度 |
