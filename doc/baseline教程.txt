项目简介
一、赛题背景介绍
本赛题聚焦构建一套“市场参与者交易行为识别与资金流向分析”的体系化解决方案，要求结合逐笔成交数据、订单簿微观结构特征、个股基本面等信息，实现从海量高频数据中自动解析不同参与者的买卖行为和资金流向，识别背后真实意图的模型——例如识别不同订单的资金属性（游资/量化私募/散户等），分析资金背后意图（吸筹、试盘、对倒、拉升、出货等），并生成清晰易懂的资金流向分析结果（当前谁占主导，买入还是卖出，真实意图是什么）。解决该问题将帮助投资者穿透复杂盘面、理性跟随不同机构资金动向，优化买卖时机，降低被游资或虚假挂单误导的风险，实现从“凭感觉跟风”到“有数据支撑的辅助决策”。
在股票投资中，市场参与者（公募基金、量化私募、游资等）的大额买卖行为往往对股价产生显著影响，其资金流向被视为重要的行情风向标。但普通投资者面对 Level-2 数据时普遍存在两大痛点：
● 信息"碎片化"：单个指标难以还原机构真实的建仓、拉抬或出货意图，机构常通过拆单、对倒等方式隐藏真实动作。
● 解读"滞后性"：多数资金流向指标基于日线统计，盘中动态变化无法捕捉，等信号明确时股价往往已大幅偏离理想买点。
通俗理解：假设你是一个散户，看到某只股票突然放量大涨。你想知道——这是游资在拉涨停准备出货，还是量化基金在程序化做 T+0 套利？如果是前者，你该跟；如果是后者，你该躲。本赛题就是训练 AI 来自动回答这个问题。
参赛者基于A股个股的逐笔委托、逐笔成交、逐笔撤单、十档盘口快照四大类全量Level2数据，构建完整的方法论体系，精细化区分游资/量化两类机构资金，识别各类市场参与者的买卖方向与交易意图，最终输出可落地的交易模式识别与资金流向分析结果。
二、任务介绍
本赛题包含两大核心任务：
Task1：交易模式识别（无监督聚类任务）
● 核心目标：基于全量Level2特征，对单日个股的交易行为进行无监督聚类，划分出具有显著区分度的交易模式，输出可解释的模式类型与业务说明；
● 核心要求：聚类结果需满足「类内高聚合、类间高区分」，模式解释需贴合A股市场真实交易逻辑，无业务常识错误；
● 输出文件：pattern_reco.csv，固定4个字段，顺序不可修改：stock_code(股票代码)、transaction_date(交易日期)、pattern_type(交易模式类别)、pattern_explanation(交易模式详细说明)。
Task2：资金类型与交易意图识别（无真值规则判别任务）
● 核心目标：基于Level2数据特征，识别单日个股的主导资金类型，同时判断资金的核心交易意图；
● 核心要求：资金类型仅可输出「游资/量化/散户」三类，交易意图仅可输出「买入/卖出/T0交易」三类，严格匹配赛事指定分类；
● 输出文件：predict_result.csv，固定4个字段，顺序不可修改：stock_code(股票代码)、transaction_date(交易日期)、capital_type(主导资金类型)、capital_intention(资金交易意图)。

提交格式要求

● 所有结果需基于当日盘中可获取数据生成，无未来函数；
评分规则
A/B榜总得分 = 交易模式识别分 × 0.4 + 参与者识别分 × 0.6，其中：
（1）Task1 交易模式识别（40%）：基于轮廓系数、CH指数、Wasserstein距离、DTW时序距离四大指标，评估聚类的类内聚合度与类间区分度；
1. 轮廓系数：衡量类内聚合度与类间区分度，取值范围[-1,1]，越接近1效果越好；
2. CH指数：衡量类间方差与类内方差的比值，值越高聚类效果越好；
3. Wasserstein距离：衡量不同交易模式的分布差异，距离越大区分度越好；
4. DTW时序距离：衡量不同交易行为的时间序列差异，距离越大区分度越好。
（2）Task2 资金类型识别【参与者识别】（60%）：基于平台真实资金标签，计算加权F1-Score，评估分类准确率。
5. 加权F1-Score：基于真实标签计算，是分类任务的核心评分指标，越接近1效果越好；
6. 精确率：预测为某类资金的样本中，真实为该类的比例，衡量查准率；
7. 召回率：真实为某类资金的样本中，被正确预测的比例，衡量查全率。
合规红线要求
（1）严禁使用未来函数：所有特征、模型、规则仅可使用当日盘中可获取的Level2数据，不得使用当日收盘后、未来交易日的任何数据；
（2）严禁硬编码个股标签：不得针对特定个股、特定日期设置固定的标签规则，所有规则需具备普适性；
（3）代码可复现性要求：B榜前15名队伍需提交完整项目代码，确保评审可完整复现结果，代码逻辑需与方案报告一致。
分析赛题、对问题进行建模
一、赛题核心分析
赛事背景
本赛题是国内金融AI领域面向全量Level2高频数据的机构资金识别专项赛事，核心解决A股市场中机构资金行为隐蔽、普通投资者无法穿透的行业痛点。本赛题希望参赛者基于A股个股的逐笔委托、逐笔成交、逐笔撤单和十档盘口快照数据（提示：可通过淘宝、闲鱼、百度网盘等渠道自行获取相关数据），构建一套完整方法论体系，对游资/量化两类机构进行精细化的区分，识别各类参与者的买卖方向、买卖意图。整体方法的技术手段不做强制要求，鼓励参赛者结合大模型等AI技术和经典量化模型给出更好的解决方案。
问题本质
本赛题的核心是无监督场景下的金融时间序列数据挖掘与行为识别问题，可拆解为两个相互耦合的子问题：
（1）Task1 无监督聚类问题：无任何标注标签，基于高维的Level2时间序列特征，对单日个股的交易行为进行分簇，核心目标是最大化类内聚合度与类间区分度，同时为每个簇赋予可解释的业务含义，自由打标，不限制个数；
（2）Task2 无真值规则判别问题：无官方提供的训练标签，基于A股量化交易的行业常识与资金行为规律，构建多维度的量化打分规则，区分游资/量化/散户三类资金，同时识别资金的交易意图，核心目标是最大化与实盘真实资金行为的匹配度。
问题的核心难点在于：
● 高频Level2数据维度高、噪声大，需要从海量数据中提取与机构交易行为强相关的有效特征；
● 机构交易行为具有极强的隐蔽性，需要穿透拆单、对倒、虚假挂撤单等操作，识别真实的资金属性与意图；
● 无任何标注标签，无法使用传统的有监督机器学习算法，所有方案需基于金融逻辑与无监督方法构建，同时保证可解释性。

二、任务分层拆解
Baseline完整开发流程可拆解为4个核心层级，每个层级有明确的目标、输入、输出与核心工作，形成完整的落地闭环：
① 数据理解层
● 目标：深入理解三源 Level-2 CSV 数据的结构和含义。
● 关键动作：
  ○ 熟悉三源CSV（行情.csv/逐笔成交.csv/逐笔委托.csv）的中文列名与业务含义，通过 HQ_COL_MAP 统一映射为英文字段。
  ○ 关键发现 1：行情.csv 的 成交量/成交额/成交笔数/当日累计成交量 均为当日累计值，但 逐笔成交.csv 的 成交数量 是逐笔值，特征工程应直接基于逐笔成交的 amount = 成交价格 × 成交数量 分级，无需 diff 累计值。
  ○ 关键发现 2：时间 字段为 HHMMSSmmm 格式整数（如 93000000=9:30:00.000），必须通过 _parse_time_to_seconds() 转换为日内秒数后做时段判定，不能用 UTC 时间戳。
  ○ 关键发现 3：逐笔成交.csv 的 BS标志（'B'买/'S'卖）直接提供主动买卖方向标签，无需像旧版基于价格变动推断；逐笔委托.csv 的 委托类型（'U'撤单/'1'成交/'0'新增）是构建 CB 撤单行为特征的核心。
② 特征工程层
● 目标：从三源高频数据中提取对交易行为和资金流向有区分度的特征。
● 关键动作：
  ○ OSS 大单分级：基于 逐笔成交.csv 的 amount 字段按超大单(≥50万)/大单(≥10万)/中单(≥4万)/小单(<4万) 分级统计金额和笔数占比。
  ○ TRD 交易结构：平均成交额/量、成交额标准差、总笔数。
  ○ RS 订单时序：成交间隔变异系数、拆单相似度、订单爆发率、高频交易占比、间隔偏度/峰度。
  ○ CB 撤单行为：基于 委托类型=='U' 计算撤单率、撤单成交比、买卖撤单分化、撤单金额占比。
  ○ AP 主动成交：基于 BS标志 计算主动买卖占比、净成交占比、连续买卖笔数、单边强度。
  ○ PI 日内时段：8个半小时时段成交占比、开盘30分钟/尾盘10分钟占比、时段集中度（基于 seconds 计算）。
  ○ PD 价格发现：从 行情.csv 提取开盘跳空、日内收益、振幅、涨跌停距离、VWAP偏离、价格反转、脉冲检测、大单方向一致性。
  ○ OBP 盘口衍生：从十档申买/申卖价量提取最优价差、盘口失衡度、大单挂单占比、加权价差统计、深度变化率。
  ○ 盘口动态：价差波动、相对价差、买卖深度比变化、净买盘变化。
  ○ ** 资金集中度**：赫芬达尔指数、Top5%大单占比、成交额熵、委托成交比、撤单聚集度、间隔自相关、买卖不对称性、连续同向笔数。
③ 模型构建层
● 目标：构建交易模式聚类和参与者识别的模型。
● 关键动作：
  ○ Task 1（4模型融合聚类）：_normalize_features_robust（50% RobustScaler + 50% Rank 百分位）后，运行 KMeans++/DBSCAN/Agglomerative/GMM 四模型多数投票融合；通过 PATTERN_RULES（10种模式 × 6特征 × 方向）贪心匹配聚类到语义模式；多样性校准将每种模式占比约束在 6%-18%。
  ○ Task 2（两阶段识别）：Stage1 用 KMeans 在10个行为特征上将全市场聚为3-6个行为组；Stage2 组内 Z-score 归一化后计算游资/量化/散户三因子加权得分（15/15/13维特征），取最高分确定资金类型；意图用 buy_signal/sell_signal 多维度（净买卖比、VWAP偏离、大单买入比、盘口失衡、日内收益）打分判定买入/卖出/T0交易。
④ 结果输出层
● 目标：按照赛题要求格式输出结果文件，并进行格式校验。
● 关键动作：
  ○ 生成 pattern_reco.csv：stock_code, transaction_date, pattern_type, pattern_explanation。
  ○ 生成 predict_result.csv：stock_code, transaction_date, capital_type, capital_intention。
  ○ 格式校验：字段顺序、合法值检查（capital_type 仅含游资/量化/散户，intention 仅含买入/卖出/T0交易）。
  ○ 编码为 UTF-8-sig，打包为 submit.zip 提交。
三、开发建议
1. 从简单开始：先跑通 Baseline，理解数据流和任务目标，再逐步优化特征和模型。
2. 重视特征工程：高频数据的特征提取是核心，建议多尝试盘口微观结构特征（OFI、价差动态、大单冲击等）。
3. 注意累计值转逐笔：volume/amount/transactions/bigordervolume 均为累计值，直接使用会导致 OSS 分类错误。
4. 注意时区问题：date 字段是 UTC 时间戳，hh 字段是北京时间小时。时段判断必须使用 hh。
5. 伪标签/规则质量：Task 2 没有真实标签，规则设计的金融逻辑合理性直接影响模型效果。
6. 代码可复现性：固定随机种子，使用相对路径，代码注释充分，确保环境可复现。
7. 善用已有字段：训练数据中很多字段（如 changepercent、rangepercent、bigordervolume）已经直接有值，直接使用比重新计算更准确。

def _parse_time_to_seconds(time_val):
    """HHMMSSmmm → 日内秒数"""
    t = np.int64(time_val)
    return (t // 10000000) * 3600 + ((t // 100000) % 100) * 60 + ((t // 1000) % 100)

def load_snapshot(stock_code, date_str, base_dir='.'):
    """加载3秒Level-2行情快照（列裁剪+dtype优化加速）"""
    fpath = os.path.join(base_dir, date_str, stock_code, '行情.csv')
    df = pd.read_csv(fpath, encoding='gbk', usecols=use_cols, dtype=dtype_map, low_memory=False)
    df = df.rename(columns=HQ_COL_MAP)
    df['seconds'] = _parse_time_to_seconds(df['time'].values)
    df['hour'] = df['seconds'] // 3600
    df['stock_code'] = stock_code
    df['transaction_date'] = date_str
    df = df.sort_values('seconds').reset_index(drop=True)
    return df

重要说明：
● 三源独立读取：行情.csv（3秒快照）、逐笔成交.csv（逐笔成交）、逐笔委托.csv（逐笔委托）分别加载，避免混合读取造成的字段错位。所有 CSV 均为 GBK 编码、中文列名，通过 HQ_COL_MAP 统一映射为英文字段。
● 时间标准化：时间 字段为 HHMMSSmmm 格式整数（如 93000000=9:30:00.000），通过 _parse_time_to_seconds() 转换为日内秒数后做时段判定（开盘30分钟 9:30-10:00、尾盘10分钟 14:50-15:00、8个半小时时段），不能用 UTC 时间戳。
● 逐笔成交直接分级：逐笔成交.csv 的 成交数量 是逐笔值，amount = 成交价格 × 成交数量 直接得到逐笔成交额，按超大单(≥50万)/大单(≥10万)/中单(≥4万)/小单(<4万) 分级，无需对 行情.csv 的累计值做 diff。
● BS标志直接可用：逐笔成交.csv 的 BS标志（'B'买/'S'卖）直接提供主动买卖方向标签，无需像旧版基于价格变动推断。
● 委托类型识别撤单：逐笔委托.csv 的 委托类型（'U'撤单/'1'成交/'0'新增）是构建 CB 撤单行为特征的核心。

    feat_mat = np.nan_to_num(df_feat[feat_cols].values.astype(float), nan=0, posinf=0, neginf=0)

    # 第1步：RobustScaler（用中位数和IQR缩放，对异常值鲁棒）
    scaler = RobustScaler()
    feat_scaled = scaler.fit_transform(feat_mat)

    # 第2步：Rank百分位特征（量纲无关）
    n, d = feat_scaled.shape
    rank_feat = np.zeros((n, d))
    for j in range(d):
        col_vals = feat_scaled[:, j]
        rank_feat[:, j] = np.argsort(np.argsort(col_vals)) / (n - 1)

    # 混合：50% RobustScaled + 50% Rank
    feat_hybrid = 0.5 * feat_scaled / (np.std(feat_scaled, axis=0) + 1e-8) + 0.5 * rank_feat
    return feat_hybrid, feat_cols
Step 2：四个聚类模型并行运行，再通过 多数投票融合 得到最终聚类标签：
def task1_multi_model_fusion(df_feat, percentiles):
    n_patterns = len(PATTERN_NAMES)  # 10种交易模式
    feat_hybrid, _ = _normalize_features_robust(df_feat)

    cluster_labels = np.zeros((n, 4), dtype=int)
    # Model 1: KMeans++（经典球形聚类）
    km = KMeans(n_clusters=n_patterns, init='k-means++', n_init=10,
                random_state=RANDOM_SEED, max_iter=300)
    cluster_labels[:, 0] = km.fit_predict(feat_hybrid)

    # Model 2: DBSCAN（密度聚类，识别任意形状和异常点）
    # 自适应 eps：用 k-distance 图 90 分位数估计
    nn = NearestNeighbors(n_neighbors=min(20, n-1)).fit(feat_hybrid)
    distances = nn.kneighbors(feat_hybrid)[0][:, -1]
    eps = max(np.percentile(distances, 90), 0.1)
    db = DBSCAN(eps=eps, min_samples=max(5, n // 200))
    cluster_labels[:, 1] = db.fit_predict(feat_hybrid)

    # Model 3: Agglomerative（层次聚类，捕捉层级结构）
    ag = AgglomerativeClustering(n_clusters=n_patterns)
    cluster_labels[:, 2] = ag.fit_predict(feat_hybrid)

    # Model 4: GMM（高斯混合，软分配）
    gmm = GaussianMixture(n_components=n_patterns, covariance_type='full',
                          random_state=RANDOM_SEED, max_iter=200)
    cluster_labels[:, 3] = gmm.fit_predict(feat_hybrid)

    # 多数投票融合（DBSCAN 噪声点 -1 过滤掉，平票时回退 KMeans）
    final_labels = np.zeros(n, dtype=int)
    for i in range(n):
        votes = cluster_labels[i, :]
        valid_votes = votes[votes >= 0]
        if len(valid_votes) == 0:
            final_labels[i] = votes[0]  # KMeans 兜底
        else:
            counter = Counter(valid_votes)
            most_common = counter.most_common()
            if len(most_common) == 1 or most_common[0][1] > most_common[1][1]:
                final_labels[i] = most_common[0][0]
            else:
                final_labels[i] = votes[0]  # 平票用 KMeans

# 模式打分：对每个特征按百分位计算方向性得分
def _score_pattern_vectorized(df_feat, pname, percentiles):
    conditions = PATTERN_CONDITIONS[pname]
    scores = np.zeros(n)
    for col, direction in conditions:
        pcts = np.array([np.mean(all_vals <= v) for v in vals])  # 跨样本百分位
        pcts = np.clip(pcts, 0.01, 0.99)
        if direction == +1:   scores += pcts                  # 正向：值越大得分越高
        elif direction == -1: scores += (1 - pcts)            # 反向：值越小得分越高
        else:                 scores += (1 - np.abs(pcts - 0.5) * 2)  # 对称
    return scores / weight_sum

五、baseline方案思路
本 Baseline的设计思路是 "三源数据驱动 + 多模型融合聚类 + 两阶段资金识别 + 多样性校准"：
1. 三源CSV独立读取：load_snapshot/load_trades/load_orders 分别读取 行情.csv/逐笔成交.csv/逐笔委托.csv，通过 HQ_COL_MAP 将中文列名统一映射为英文字段，避免混合读取造成的字段错位。
2. 时间标准化：时间 字段为 HHMMSSmmm 格式整数（如 93000000=9:30:00.000），通过 _parse_time_to_seconds() 转换为日内秒数后做时段判定（开盘30分钟/尾盘10分钟/8个半小时时段），不能用 UTC 时间戳。
3. 特征工程10大类约100维：基于三源数据提取 OSS大单分级/TRD交易结构/RS订单时序/CB撤单行为/AP主动成交/PI日内时段/PD价格发现/OBP盘口衍生 共8类官方特征，并补充 盘口动态（价差波动/相对价差/买卖深度比变化/净买盘变化）与 资金集中度（赫芬达尔指数/Top5%大单占比/成交额熵/委托成交比/撤单聚集度/间隔自相关/买卖不对称性/连续同向笔数）2类衍生特征。
4. 逐笔成交直接分级：逐笔成交.csv 的 成交数量 是逐笔值，特征工程直接基于 amount = 成交价格 × 成交数量 按超大单(≥50万)/大单(≥10万)/中单(≥4万)/小单(<4万) 分级，无需对 行情.csv 的累计值做 diff。
5. 4模型融合聚类：Task1 采用 KMeans++/DBSCAN/Agglomerative/GMM 四模型多数投票融合（k=10），比单一 KMeans 更鲁棒，能综合密度/层次/概率/质心四类聚类思想的优势。
6. PATTERN_RULES贪心匹配：定义 10 种交易模式 × 6 个特征 × 方向（+1/-1/0）的规则矩阵，对每个聚类簇的特征均值做贪心匹配，命中数最多的模式即为该簇的语义标签，实现数学聚类到业务模式的可解释映射。
7. 两阶段资金识别：Stage1 用 KMeans 在10个行为特征上将全市场聚为3-6个行为组；Stage2 组内 Z-score 归一化后计算游资(15维)/量化(15维)/散户(13维) 三因子加权得分，取最高分确定资金类型，比全局打分更能消除行业/市值差异。
8. 多维度意图打分：buy_signal/sell_signal 各7维信号（净买卖比/VWAP偏离/大单买入比/盘口失衡/日内收益/尾盘动量/开盘强度）加权打分（权重1.0/0.7/0.6/0.5/0.4/0.3/0.3），两信号均弱则判定为 T0交易，专为量化高频套利场景设计。
9. 多样性校准：Task1 模式分布约束在 6%-18%，Task2 资金类型约束在 22%-45%，意图中买入/卖出约束在 18%-40% 且 T0≥12%，避免单一类别主导输出，提升与真实市场分布的匹配度。
10. 量纲鲁棒处理：Task1 采用 50% RobustScaler + 50% Rank 百分位混合归一化（_normalize_features_robust），Task2 采用组内 Z-score 归一化 + min-max 归一化，分别应对不同聚类算法对量纲的敏感性。
11. 边界处理鲁棒：DBSCAN 噪声点回退至 KMeans 标签、Task2 少样本时调用 _task2_rule_only_fallback 纯规则兜底、np.nan_to_num 处理 NaN/Inf、assert 校验输出合法性，确保全流程稳定运行。
六、baseline核心逻辑
1. 数据安全逻辑：所有特征、规则、模型均基于三源 Level2 当日盘中可获取数据，全局固定 RANDOM_SEED=42，无未来函数、无跨日数据、无硬编码标签，完全符合赛事合规要求；
2. Task1核心逻辑：4模型融合聚类（KMeans/DBSCAN/Agglomerative/GMM 多数投票，k=10）→ PATTERN_RULES 贪心匹配 10 种交易模式（大单吸筹/压单吸货/尾盘突袭/集合竞价异动/对倒拉升/盘中诱多/分时脉冲/涨停板打开/连续小单推升/日内套利）→ 多样性校准将每种模式占比约束在 6%-18%，确保聚类结果的区分度与业务可解释性；
3. Task2核心逻辑：两阶段识别——Stage1 KMeans 行为组聚类(3-6组) → Stage2 组内 Z-score 归一化后游资(15维)/量化(15维)/散户(13维) 三因子加权打分，游资侧重大额成交/单边买入/盘口买盘失衡，量化侧重小单高频/拆单均匀/窄价差/多空均衡/撤单密集，散户侧重小单分散/无方向性/撤单率低；
4. 意图识别核心逻辑：buy_signal/sell_signal 各7维信号加权打分（权重1.0/0.7/0.6/0.5/0.4/0.3/0.3），覆盖主动成交方向/价格冲击/盘口失衡/日内收益/尾盘动量/开盘强度六维度，两信号均弱则判定为 T0交易，与游资单边趋势性交易形成有效区分；
5. 双任务联动逻辑：Task1 与 Task2 共享 df_feat 特征矩阵，Task1 的 PATTERN_RULES 与 Task2 的资金/意图判定独立运行，但最终在 _post_filter_calibrate 中对目标股票子集统一做后处理校准，确保整体分布合理性；
6. 可解释性逻辑：10 种交易模式均有明确金融含义，游资/量化/散户三因子均有对应特征组支撑，buy/sell_signal 7 维信号均可追溯，无黑箱模型，符合金融场景的可解释性要求，提升方案的评审得分。
